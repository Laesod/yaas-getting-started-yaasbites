README

1 Goal

In this Bite we want to start using micro-services that are available from the Yaas Market Place. In particular we will modify our persistence logic to use the Document Service, rather than in-memory logic with slick and h2. If you browse the Yaas Market Place, you will find the Document Service in the Persistence (Beta) Package. Its documentation will tell you that callers to this service require 3 scopes to successfully invoke it: hybris.documentview, hybris.documentmanage, and hybris.documentadmin.

2 Changes needed in the Yaas Project and the microservice

2.1 in the new yaas project

2.1.1 subscribe to the Persistence (Beta) Package, so that the API Proxy will (once we have a suitable access-token), allow us to call the Document Service

2.1.2 register our microservice in YaaS

2.1.3 add a YaaS Client to our YaaS Service, that has the rights to request and be given the new scopes.

2.2 add logic in the microservice to accquire access-tokens and invoke Document Service

2.2.1 we will have 2 new classes

  1. services.DocServiceWrapper
  2. services.OauthService

3 Steps

3.1 cd to project-directory scala-yaasbite800

3.2 adjust the name in the manifest.yml to make it unique

3.3 Determine where the microervice will be deployed in the cloud: We need the location of our microservice in the cloud, but it is not yet ajusted and ready to be deployed.

3.3.1 to get around this, we will skip running tests for now

3.3.2 run the following commands

  1. activator dist
  2. cf push
  3. cf apps take a note of the URL(1) to where the package has been deployed

3.4 in your Yaas project

3.4.1 Subscribe to the Persistence Package: Add a subscription to the Persistence (Beta) Package

3.4.2 Wire the microservice into YaaS

  1. Create a new Service in the YaaS Builder and specify the deployed URL (1) in the service's Source URL field
  2. Be sure you use the https prefix (protocol) in the Source URL field

3.4.3 Add a YaaS Client to your YaaS Service, and add the three new scopes to its list of Required Scopes.

  1. This will allow us to use it's credentials to acquire access-tokens with those scopes

3.4.4 Be sure to deploy the YaaS Service with the Deploy Button in the YaaS Service's page(it is not enough just to wire it up)

3.5 change configurations

3.5.1 In your Micro-service: Search for NEEDSADJUSTING and modify those entries to match your YaaS Project settings

3.6 try running the tests

3.6.1 activator clean compile test

3.6.2 If these pass, your service is succesfully calling the Deployent Service.

3.6.3 Debug through the tests to find how the access-tokens are acquired, and how the document service is being called.

3.7 Repackage and redeploy

3.7.1 activator dist

3.7.2 cf push

3.8 Try hitting your website. You should see that the application works as before but is now using the Document Service for its persistence.
